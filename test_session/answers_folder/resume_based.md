## Resume-Based Questions

**1. In your "Late Fusion Multimodal ML for Skin Cancer Classification" project, you achieved 98.11% accuracy. Can you walk me through the decision-making process for choosing a late fusion architecture over early or intermediate fusion, and what specific challenges you encountered when integrating the outputs of the CNN and RFC models?**
*Model Answer:* We chose late fusion because our data modalities—images processed by a CNN and structured clinical metadata processed by a Random Forest—are fundamentally different. Early fusion by concatenating raw features would have created a very high-dimensional, sparse space that's difficult for a single model to learn effectively. Late fusion allowed each model to learn specialized representations first. The main challenge was integrating the outputs; the CNN produced probability distributions while the RFC produced class predictions. We experimented with simple averaging and voting before settling on a Gradient Boosting model as a meta-learner to intelligently weigh the predictions from both models, which proved most effective at capturing their complementary strengths.

**2. Your "SuperGit" project uses LLMs for an automated CI/CD pipeline. Could you elaborate on the specific tasks the LLM performs, such as code generation or review? How did you engineer your prompts and what measures did you implement to handle potential inaccuracies or security vulnerabilities from the LLM-generated code?**
*Model Answer:* In SuperGit, the LLM primarily performs two tasks: generating boilerplate code based on issue descriptions and providing initial code reviews on pull requests. For prompt engineering, we provided a structured context including the file to be modified, surrounding code snippets, and a clear natural language instruction. To handle inaccuracies, LLM suggestions were never auto-committed. Instead, they were posted as comments for human review. For security, we ran a static analysis tool as a subsequent step in the GitHub Actions pipeline to scan any LLM-suggested code for common vulnerabilities before a developer could approve the changes.

**3. Developing a Quantum Convolutional Neural Network is a highly specialized task. What was the most significant conceptual or implementation challenge you faced when working with Pennylane, and how did you reason about the trade-offs between a QCNN and a classical CNN for the Fashion MNIST dataset?**
*Model Answer:* The most significant challenge was designing the quantum circuit, specifically the data encoding layer that maps classical image pixel data into the quantum state space. It's a non-trivial task that heavily influences performance. The trade-off was clear: a classical CNN is mature, scalable, and guaranteed to perform well on a dataset like Fashion MNIST. The QCNN, while achieving a respectable 84.5%, was primarily an exploration into a novel computational paradigm. The goal wasn't to outperform the classical CNN but to investigate the potential of quantum circuits for feature extraction and understand the current limitations of quantum machine learning simulators.

**4. During your UI/UX internship at Enyard, you designed the "Booba" taxi app. Describe a specific instance where user research or usability testing feedback led to a significant pivot in your initial design for a key user flow, like the booking process or payment screen.**
*Model Answer:* In my initial design for the Booba app, the booking flow was a three-step process: enter destination, select car type, then confirm payment details. However, during usability testing, we observed users wanted to see the estimated fare and car availability much earlier. This feedback led to a significant pivot. We redesigned it into a single-screen flow where as soon as the user enters a destination, the map updates to show available cars, and a bottom sheet displays all car types with their estimated fares upfront. This change drastically reduced friction and improved the perceived transparency of the booking process.

How do you believe these two disciplines inform each other in your work, and can you provide an example from a project where your understanding of user experience directly influenced your approach to building an ML model or system?**
*Model Answer:* I believe UI/UX defines the "what" and "why" of a problem from a human perspective, while ML provides a powerful "how" to solve it. My UI/UX background ensures I'm building a system that's not just technically impressive but also genuinely useful and interpretable. For example, in the skin cancer classification project, a purely technical goal is high accuracy. However, my UX mindset made me consider the end-user: a dermatologist. They need more than a "cancer/no cancer" label; they need confidence scores and interpretability. This directly influenced our model's output, pushing us to implement methods to visualize which parts of the image the CNN focused on, making the AI's decision more trustworthy for the user.
