**1. In your "Late Fusion Multimodal ML for Skin Cancer Classification" project, you achieved 98.11% accuracy. Can you walk me through the "late fusion" technique you used and explain the specific challenges you faced when integrating image data with tabular metadata?**
*Model Answer:* Late fusion, in this context, involved training two separate models: a CNN for the image data and a deep neural network for the tabular metadata. Each model produced a feature vector, or a set of predictions. We then concatenated these outputs and fed them into a final, simple neural network that made the ultimate classification. The main challenge was balancing the influence of the two data types. Initially, the numerical features from the metadata dominated the feature vector from the images. We solved this by applying feature scaling and standardization to ensure both modalities contributed effectively to the final prediction, which was key to achieving the 98.11% accuracy.

Specifically, how did you integrate the LLM with GitHub Actions, and what prompts or fine-tuning did you use to ensure the generated code was both functional and optimized?**
*Model Answer:* The architecture is event-driven. A GitHub Actions workflow is triggered on a push or pull request. This workflow runs a Python script that takes the code changes as input. The script then calls the OpenAI API, sending the code to a GPT model. For integration, we used API requests from the Python script running in the GitHub runner. The key was prompt engineering. For code review, the prompt instructed the LLM to act as a senior developer, checking for things like PEP 8 compliance, potential null pointer exceptions, and inefficient loops. For generation, we provided clear context and examples in the prompt to ensure the output was not only functional but also aligned with the project's existing style.

**4. Your project on Quantum Convolutional Neural Networks (QCNN) is quite advanced. What motivated you to explore quantum computing for a classic problem like image classification, and what were the key differences in implementation between the QCNN using Pennylane and a traditional CNN using TensorFlow/Keras?**
*Model Answer:* My motivation was to explore how emerging technologies like quantum computing could potentially offer advantages over classical methods, especially in complex pattern recognition tasks. The key implementation difference was in the feature extraction layers. In a traditional CNN, I used TensorFlow's convolutional and pooling layers. For the QCNN, I used Pennylane to design a quantum circuit with rotational and CNOT gates to act as the "convolutional" layer. This quantum circuit processes data in a high-dimensional Hilbert space, which theoretically allows it to capture more complex features than classical layers. The output from the quantum layer was then passed to classical dense layers for the final classification.
