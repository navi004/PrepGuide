## Technical Questions

**1. Explain the difference between a B-Tree and a Hash Map for indexing a database. In what scenarios would you choose one over the other?**
*Model Answer:* A B-Tree is a self-balancing tree structure that stores data sorted, making it efficient for range queries and sequential access. A Hash Map uses a hash function to map keys directly to storage locations, providing O(1) average time complexity for direct lookups. I would choose a B-Tree index for a column that is often used in range queries, like finding all transactions between two dates. I would choose a Hash Map index for a column that requires very fast, exact-match lookups, such as retrieving a user by their unique `user_id`.

**2. Describe the principles of RESTful APIs. What does it mean for an API endpoint to be idempotent, and can you give an example of an idempotent HTTP method versus a non-idempotent one?**
*Model Answer:* The core principles of REST are a client-server architecture, statelessness (each request contains all info needed), cacheability, a layered system, and a uniform interface using standard HTTP methods. An operation is idempotent if making the same request multiple times has the same effect as making it once. A good example of an idempotent method is `PUT /users/123` to update a user's profile; running it multiple times results in the same final state. A non-idempotent method is `POST /users` to create a new user, as running it multiple times will create multiple new users.

**3. You are given a large, unsorted log file where each line contains a timestamp and a message. How would you efficiently find the top K most frequent error messages that occurred within a specific time window? Describe the data structures and algorithms you would use and analyze the time complexity.**
*Model Answer:* I would iterate through the file once. For each line, I'd parse the timestamp and check if it's within the target window. If it is, I'd use a Hash Map to store the frequency of each error message, with the message as the key and its count as the value. After processing the relevant lines, I'd use a Min-Heap of size K to find the top K elements. I'd iterate through my hash map, adding elements to the heap. If the heap is full and a new message's frequency is higher than the heap's minimum, I'd replace the minimum. The overall time complexity would be O(N log K), where N is the number of log entries in the time window.

**4. What is the difference between a microservices architecture and a monolithic architecture? Discuss two potential benefits and two potential drawbacks of choosing a microservices approach for a large-scale enterprise application.**
*Model Answer:* A monolithic architecture is a single, unified application unit. A microservices architecture breaks the application into a collection of small, independently deployable services that communicate over a network.
*   **Benefits:** 1) **Independent Scalability:** Each service can be scaled independently based on its specific load, which is more resource-efficient. 2) **Technology Flexibility:** Teams can choose the best technology stack for their particular service without affecting others.
*   **Drawbacks:** 1) **Operational Complexity:** Managing, monitoring, and deploying dozens of services is much more complex than a single monolith. 2) **Distributed System Challenges:** You have to deal with issues like network latency, fault tolerance, and eventual data consistency between services.

**5. In Java, what is the difference between an `interface` and an `abstract class`? Provide a practical example of when you would choose to use one over the other.**
*Model Answer:* An `abstract class` can have instance variables, constructors, and both abstract and implemented methods; a class can only extend one abstract class. An `interface` is a contract of methods that a class must implement; a class can implement multiple interfaces. I would use an `abstract class` to create a base with shared state and functionality, like an `abstract class Vehicle` with properties like `speed` and a method `accelerate()`, which `Car` and `Truck` would extend. I would use an `interface` to define a capability, like a `Serializable` interface, which completely unrelated classes like `User` and `Configuration` could both implement.

**3. A critical API endpoint in your application, which retrieves user profile data from a MySQL database, has become very slow. Users are reporting long load times. What are the first five steps you would take to systematically diagnose the root cause of this performance issue?**
*Model Answer:* My first five steps would be:
1.  **Check Monitoring and Logs:** I'd start by looking at our Application Performance Monitoring (APM) tool and application logs to identify if the latency is in the application code, the database query, or network I/O. I'd look for slow query logs specifically.
2.  **Analyze the SQL Query:** I'd isolate the exact SQL query being executed. Using the `EXPLAIN` command in MySQL, I would analyze the query's execution plan to see if it's using proper indexes or performing a slow full-table scan.
3.  **Verify Database Indexes:** Based on the `EXPLAIN` plan, the most likely culprit is a missing index. I would verify that the columns used in the `WHERE` clause and `JOIN` conditions of the query are indexed.
4.  **Investigate for N+1 Problems:** I'd review the application code for the endpoint to check for the "N+1 query" anti-pattern, where the code retrieves a list of items and then makes an additional database call inside a loop for each item.
5.  **Evaluate Caching Strategy:** If the data is read-heavy and doesn't change frequently, I'd assess if a caching layer like Redis is in place. If not, implementing caching for user profiles would be my top recommendation to avoid hitting the database on every request.
