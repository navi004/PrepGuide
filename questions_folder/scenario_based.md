## Technical Scenario-Based Questions

1. You are tasked with deploying your skin cancer detection model as a web service that can handle thousands of requests per day. Outline the high-level architecture you would propose, mentioning the key components like the web server (e.g., using Flask), the model hosting, and how you would handle user image uploads.
2. A user reports that your "SuperGit" CI/CD pipeline is intermittently failing during the automated testing stage, but it works fine when you run the tests manually. What are the first five steps you would take to investigate and debug this issue?
3. Imagine a client has a web application with a feature that loads user profile data from a database, and it's become very slow as the user base has grown. What potential bottlenecks would you investigate, and what are some strategies you might implement to improve the performance?
4. You are given a 10 GB CSV file containing user activity logs with columns like `timestamp`, `user_id`, `action`, and `details`. The file is too large to fit into memory. How would you use Python and libraries like Pandas to efficiently calculate the number of unique users who performed a 'login' action each day?
5. A Deloitte client from the retail industry wants to build a "customer loyalty application" but has only a vague idea of the requirements. They just know they want to "reward good customers." What key questions would you ask the client to clarify the project's scope, define the core features, and understand the technical constraints before you start designing the solution?
